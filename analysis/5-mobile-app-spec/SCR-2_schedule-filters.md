## Обзор

Панель настройки параметров фильтрации списка тренировок. Открывается поверх
[SCR-1](./SCR-1_schedule-list.md) как bottom sheet — принято как рабочее
архитектурное решение для MVP (открытый вопрос исходного документа: модалка
vs отдельный экран — решение может быть скорректировано дизайнером).
Точный перечень фильтров не был зафиксирован в FR-4; по итогам согласования
(см. `00-OPEN-QUESTIONS-LOG.md`, вопрос №2) для MVP принят сокращённый набор
из двух параметров — диапазон дат и наличие свободных мест — чтобы не
увеличивать сложность разработки при ограниченном сроке (BR-12). Остальные
поля (время суток, инструктор), хотя и поддерживаются на уровне
`GET /trainings` в `openapi.yaml`, в MVP-версии интерфейса не используются
и остаются кандидатами для v2.

### User Story

> Как клиент, я хочу задать диапазон дат и включить показ только тренировок
> со свободными местами, чтобы быстрее найти подходящую тренировку.

### Бизнес-ценность

- Сокращает время поиска подходящей тренировки в длинном списке.
- Снижает долю "пустых" просмотров карточек без свободных мест.

## Используемые запросы

> Экран фильтров сам не выполняет запрос на получение тренировок — фильтры
> лишь формируют query-параметры для `GET /trainings`, который выполняется
> на SCR-1 после нажатия «Применить» (см. [SCR-1 → GET /trainings](./SCR-1_schedule-list.md#get-trainings)).
> Отдельного запроса на этом экране, зафиксированного в требованиях, нет.

## Макет экрана

### Структура

```
┌─────────────────────────────────────┐
│  Фильтры                       [✕]  │  ← Header bottom sheet
├─────────────────────────────────────┤
│  Дата / диапазон дат                │
```

### Компоненты

| Компонент | Описание | Обязательность |
|-----------|----------|-----------------|
| Селектор даты/диапазона | Календарь или предустановленные пресеты ("Сегодня", "Завтра", "На неделе") | Да |
| Переключатель «Только со свободными местами» | Toggle | Да |
| Кнопки "Сбросить" / "Применить" | Fixed footer | Да |

> Селекторы «Время суток», «Формат тренировки» и «Инструктор» исключены из
> MVP-набора (см. историю изменений, релиз 0.1.1) — кандидаты для v2.

## Элементы экрана

### 1. Параметры фильтрации

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| Дата / диапазон дат | Выбор одной даты или диапазона | `currentFilters.date_from/date_to` | `date_from ≤ date_to` | Обновляет локальное черновое состояние фильтров |
| Только со свободными местами | Toggle | `currentFilters.only_available` | — | Обновляет черновое состояние |

**Логика:**
- Изменения полей не отправляют запрос немедленно — накапливаются в локальном черновом состоянии до нажатия «Применить».

### 2. Кнопки действий

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| Кнопка «Сбросить» | Второстепенная кнопка | — | — | Очищает черновое состояние → закрывает шторку с `filters = {}` |
| Кнопка «Применить» | Primary button | — | — | Закрывает шторку, передаёт черновое состояние на SCR-1 как `filters` |

**Условия доступности:**
- Кнопка «Сбросить» неактивна/скрыта, если все поля уже пусты.
- Кнопка «Применить» активна всегда (в т.ч. без выбранных значений — эквивалентно показу всего расписания).

## Состояния экрана

### Таблица состояний

| Состояние | Условие | Отображение |
|-----------|---------|-------------|
| По умолчанию | `currentFilters` пуст | Все поля не заполнены |
| Восстановлено | `currentFilters` содержит значения | Поля предзаполнены ранее выбранными значениями |
| Изменено | Пользователь изменил хотя бы одно поле | Визуальное выделение изменённых значений, активна кнопка «Сбросить» |

### Диаграмма переходов

```mermaid
stateDiagram-v2
    [*] --> Default : onOpen, currentFilters пуст
    [*] --> Restored : onOpen, currentFilters заполнен
    Default --> Modified : Изменение поля
    Restored --> Modified : Изменение поля
    Modified --> [*] : Применить / Сбросить / Закрыть без изменений
```

## Критерии приёмки

### Позитивные сценарии

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-001 | **Дано** клиент выбрал формат «Для новичков» и нажал «Применить», **Когда** происходит переход на SCR-1, **Тогда** список показывает только тренировки этого формата | P0 |
| AC-002 | **Дано** ранее были применены фильтры, **Когда** клиент повторно открывает SCR-2, **Тогда** поля предзаполнены ранее выбранными значениями | P1 |
| AC-003 | **Дано** ни один фильтр не выбран, **Когда** клиент нажимает «Применить», **Тогда** список на SCR-1 показывает все тренировки без ограничений | P1 |

### Негативные сценарии

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-N01 | **Дано** выбрана дата "с" позже даты "по", **Когда** клиент пытается применить фильтр, **Тогда** отображается ошибка валидации диапазона | P1 |

### Граничные условия

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-E01 | **Дано** клиент изменил фильтры, но закрыл шторку крестиком, **Когда** он возвращается на SCR-1, **Тогда** список не изменился (черновые изменения не сохранены) | P2 |
| AC-E02 | **Дано** перезапуск приложения, **Когда** клиент снова открывает SCR-1, **Тогда** ранее применённые фильтры сброшены (для MVP не персистентны) | P2 |
