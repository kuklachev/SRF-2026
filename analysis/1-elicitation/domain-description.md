# Описание домена

## Бизнес

«Вертикаль» — частный скалодром, работающий третий год. Основной вид деятельности — проведение групповых тренировок по скалолазанию для начинающих и опытных спортсменов.

Тренировки проходят в специально оборудованном помещении и проводятся по заранее сформированному расписанию. Каждое занятие длится около 1,5 часов и проводится одним инструктором. В настоящий момент на скалодроме работают пять инструкторов, а в период высокой загрузки дополнительно привлекаются временные сотрудники.

В настоящее время запись клиентов осуществляется вручную через Telegram и бумажный журнал. Из-за этого возникают двойные бронирования, ошибки при распределении мест и высокая нагрузка на владельца.

Цель проекта — предоставить клиентам возможность самостоятельно записываться на тренировки через мобильное приложение, используя существующую серверную инфраструктуру.

---

## Процесс

Владелец заранее формирует расписание тренировок примерно на неделю вперёд через существующую административную систему.

Для каждой тренировки задаются:

- дата и время начала;
- продолжительность;
- формат занятия;
- назначенный инструктор;
- стоимость;
- количество доступных мест.

Клиент просматривает доступные тренировки, выбирает подходящую и оформляет запись.

Во время записи клиент указывает, будет ли использовать собственное снаряжение или комплект, предоставляемый скалодромом.

Один клиент может оформить запись сразу на нескольких человек (себя и до двух гостей), если имеются свободные места.

До начала тренировки клиент может отменить запись. Если отмена произведена заранее (не менее чем за два часа), место освобождается для других клиентов. Поздняя отмена фиксируется системой, однако денежных штрафов в первой версии приложения не предусмотрено.

После завершения тренировки клиент может поставить инструктору оценку по пятибалльной шкале и, при желании, оставить текстовый отзыв.

При отмене тренировки скалодромом (например, по причине профилактики или проверки оборудования) бронь клиента сохраняется, получает статус **«Отменена скалодромом»**, а клиент получает push-уведомление.

# Цель клиентского приложения

Клиентское мобильное приложение предназначено для самостоятельной записи клиентов на тренировки.

Приложение позволяет:

- просматривать расписание тренировок;
- фильтровать занятия;
- просматривать подробную информацию о тренировке;
- записываться на тренировку;
- выбирать использование собственного или прокатного оборудования;
- записывать нескольких участников одной бронью;
- просматривать свои записи;
- отменять записи;
- получать push-уведомления;
- оценивать инструкторов после посещения тренировки.

Управление расписанием, инструкторами и тренировками выполняется существующей инфраструктурой через API.

# Ключевые объекты домена

## Тренировка (слот)

Конкретное групповое занятие.

Содержит:

- дату;
- время начала;
- продолжительность;
- формат;
- инструктора;
- стоимость;
- максимальное количество мест;
- количество свободных мест.

Создаётся существующей системой и доступно клиентскому приложению только для чтения.

---

## Формат тренировки

Тип занятия.

В системе используются два формата:

- тренировка для новичков;
- тренировка для опытных спортсменов.

Формат определяет допустимое количество участников.

---

## Инструктор

Сотрудник скалодрома, проводящий тренировку.

В мобильном приложении отображается только справочная информация об инструкторе.

---

## Снаряжение

Комплект оборудования, предоставляемый скалодромом.

На этапе MVP клиент может выбрать один из вариантов:

- собственное оборудование;
- полный комплект оборудования напрокат.

Контроль доступности прокатного оборудования выполняется через существующий backend.

---

## Бронь

Запись клиента на конкретную тренировку.

Бронь содержит:

- тренировку;
- количество участников;
- использование собственного или прокатного оборудования;
- статус;
- статус оплаты.

---

## Клиент

Пользователь мобильного приложения.

Клиент может:

- просматривать расписание;
- оформлять записи;
- отменять записи;
- просматривать историю своих бронирований;
- получать уведомления;
- оставлять оценки инструкторам.

---

# Пользователи и роли

В рамках текущей разработки реализуется только роль **Клиент**.

Роли администратора и инструктора уже реализованы в существующей инфраструктуре и находятся вне текущего проекта.

| Роль | Описание | Цели | Входит в скоуп |
|------|----------|------|----------------|
| Клиент | Посетитель скалодрома | Просматривать расписание, записываться, отменять записи, получать уведомления, оценивать инструкторов | Да |
| Инструктор | Проводит тренировки | Использует существующую систему | Нет |
| Администратор (владелец) | Управляет расписанием, тренировками и инструкторами | Использует существующую административную систему | Нет |

# Ограничения

## Бизнес-ограничения

- Максимальное количество участников обычной тренировки — **16 человек**.
- Максимальное количество участников тренировки для новичков — **8 человек**.
- Один клиент может оформить запись максимум на трёх участников.
- Контроль количества свободных мест выполняется существующим backend.
- Прокатное оборудование учитывается отдельно от свободных мест.
- Бесплатная отмена возможна не позднее чем за **2 часа** до начала тренировки.
- Поздняя отмена фиксируется без применения денежных штрафов.
- Оплата осуществляется офлайн (наличные или перевод).
- Оценить инструктора можно только после завершённой тренировки.
- При отмене тренировки скалодромом бронь получает статус **«Отменена скалодромом»**.
- Клиент получает push-уведомление об отмене.
- Повторная запись на отменённую тренировку запрещена.

---

## Технические ограничения

- Разрабатывается только клиентское мобильное приложение.
- Источником данных является существующий backend.
- Все данные поступают через API.
- Клиентское приложение не создаёт и не изменяет тренировки.
- Проверка свободных мест выполняется сервером.
- Источником истины является существующий backend.

---

## Ограничения проекта

- Срок разработки — около двух месяцев.
- Бюджет ограничен.
- Реализуется MVP.
- В текущую поставку входят только мобильное приложение клиента и API.
- Интерфейсы администратора и инструктора находятся вне текущего проекта.
