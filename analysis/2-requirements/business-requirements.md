# Бизнес-требования

Этап 2. Определение бизнес-требований.

Документ определяет цели проекта, ожидаемый бизнес-результат, метрики успеха и ограничения верхнего уровня.

Скоуп проекта ограничен клиентским мобильным приложением и API для него. Управление расписанием, инструкторами и административные функции обеспечиваются существующей серверной инфраструктурой и не входят в текущую поставку.

# Метрики успеха

Целевые значения являются ориентировочными для первой версии продукта (MVP) и могут уточняться после запуска системы.

| ID | Метрика | Целевой ориентир | Связь с требованиями |
|----|----------|------------------|----------------------|
| M-1 | Доля самостоятельных записей через мобильное приложение | ≥ 70 % всех записей через 6 месяцев после запуска | BR-2, BR-3 |
| M-2 | Количество случаев двойного бронирования | 0 случаев | BR-1 |
| M-3 | Доля клиентов, самостоятельно отменяющих запись через приложение | ≥ 90 % отмен выполняются без участия владельца | BR-5 |
| M-4 | Доля клиентов, использующих мобильное приложение для просмотра расписания | ≥ 80 % зарегистрированных клиентов | BR-4 |
| M-5 | Готовность продукта к запуску | Выпуск MVP не позднее чем через 2 месяца | BR-12 |

# Ограничения верхнего уровня

## Ограничения проекта

- Проект реализуется в формате **MVP**.
- Срок разработки составляет около **двух месяцев**.
- Бюджет проекта ограничен.
- В текущую поставку входят только клиентское мобильное приложение и API.

**Источник:** Описание домена → **Ограничения проекта**.

---

## Технические ограничения

- Источником данных является существующий backend.
- Клиентское приложение получает все данные через API.
- Создание и изменение тренировок осуществляется существующей системой.
- Проверка свободных мест выполняется серверной частью.

**Источник:** Описание домена → **Технические ограничения**.

---

## Бизнес-ограничения

- Максимальная вместимость обычной тренировки — **16 участников**.
- Максимальная вместимость тренировки для новичков — **8 участников**.
- Один клиент может оформить запись максимум на **трёх участников**.
- Бесплатная отмена возможна не позднее чем за **2 часа** до начала тренировки.
- Оплата осуществляется только офлайн.
- Оценить инструктора можно только после завершения тренировки.
- При отмене тренировки скалодромом клиент получает push-уведомление, а бронь получает соответствующий статус.

**Источник:** Описание домена → **Ограничения**.

---

## Границы проекта

В текущий скоуп входят:

- регистрация клиента;
- просмотр расписания;
- просмотр информации о тренировках;
- запись на тренировку;
- запись нескольких участников;
- выбор собственного или прокатного снаряжения;
- просмотр своих бронирований;
- отмена записи;
- push-уведомления;
- оценка инструкторов.

За пределами текущего проекта находятся:

- административная панель;
- управление расписанием;
- управление инструкторами;
- создание тренировок;
- онлайн-оплата;
- программа лояльности;
- SMS- и Email-уведомления.

**Источник:** Описание домена → **Границы скоупа**.
