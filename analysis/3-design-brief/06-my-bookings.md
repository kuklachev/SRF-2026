# SCR-6. Мои бронирования

## Назначение экрана
Позволяет клиенту видеть все свои бронирования и их текущий статус, чтобы
контролировать предстоящие тренировки.

## Роль пользователя
Клиент.

## Предусловия
- Клиент авторизован в приложении (см. открытый вопрос по регистрации в
  реестре экранов).

## Точки входа
- Нижняя навигация / пункт меню «Мои бронирования».
- Переход с SCR-5 после создания брони.
- Переход из push-уведомления об отмене тренировки (SCR-8).

## Состав информации на экране
Список бронирований клиента, для каждой записи отображается (FR-12):
- тренировка (дата, время, инструктор);
- количество участников;
- выбранный вариант оборудования;
- статус брони (например: «Подтверждена», «Поздняя отмена», «Отменена
  клиентом», «Отменена скалодромом», «Завершена»);
- статус оплаты.

Список должен разделять или явно выделять предстоящие и завершённые/
отменённые бронирования (порядок и группировка — на усмотрение дизайнера,
явных требований к сортировке нет).

## Действия пользователя
- Просмотр списка бронирований.
- Тап по брони → переход на SCR-7 (детали / отмена).
- Для завершённых тренировок — переход к оценке инструктора (SCR-9), если
  оценка ещё не оставлена (FR-22).

## Состояния экрана
- **Загрузка**: индикатор ожидания данных от backend.
- **Пустое состояние**: у клиента ещё нет бронирований — предложение перейти
  к расписанию (SCR-1).
- **Ошибка**: backend недоступен — сообщение с возможностью повтора.

## Ограничения и особые случаи
- Клиент видит только собственные бронирования, доступ к чужим данным
  исключён (NFR-9).
- Данные должны быть актуальными на момент отображения (NFR-6).

## Связанные требования
- Бизнес-требования: BR-9.
- Функциональные требования: FR-11, FR-12.
- Пользовательские истории: US-7.
- Нефункциональные требования: NFR-9, NFR-6.
- Use Case: UC-5.

## Открытые вопросы для дизайна
- Нужна ли фильтрация/сортировка бронирований (например, по дате или
  статусу)? В требованиях не описана как обязательная — можно предусмотреть
  простое разделение «Предстоящие / Прошедшие» без отдельного экрана
  фильтров.
