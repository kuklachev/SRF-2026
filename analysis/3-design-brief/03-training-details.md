# SCR-3. Карточка тренировки (подробная информация)

## Назначение экрана
Отображает клиенту полную информацию о выбранной тренировке перед принятием
решения о записи.

## Роль пользователя
Клиент.

## Предусловия
- Клиент выбрал тренировку из списка (SCR-1).

## Точка входа
- Тап по карточке тренировки на SCR-1.

## Состав информации на экране
- дата, время начала, продолжительность тренировки;
- формат (обычная / для новичков), от которого зависит максимальная
  вместимость (16 или 8 участников соответственно — бизнес-ограничения);
- инструктор: имя, фото, справочная информация (без возможности перехода к
  редактированию — только просмотр, FR-3 через UC-2);
- стоимость тренировки;
- количество свободных мест из общей вместимости;
- информация о доступности проката оборудования (справочно, из backend —
  FR-9, отображается здесь либо переносится в форму записи SCR-4 — решает
  дизайнер, но данные должны быть видны клиенту до подтверждения записи);
- политика отмены (бесплатная отмена не позднее чем за 2 часа — как
  информационная справка, чтобы клиент принимал решение осознанно).

## Действия пользователя
- Кнопка «Записаться» → переход на SCR-4 (доступна только если есть
  свободные места).
- Возврат к списку тренировок (SCR-1).

## Состояния экрана
- **Загрузка**: индикатор ожидания подробных данных.
- **Мест нет**: кнопка «Записаться» неактивна / заменена статусом «Мест
  нет», запись невозможна (FR-8).
- **Тренировка отменена скалодромом**: если пользователь попал на карточку
  тренировки, впоследствии отменённой, экран должен отображать статус
  «Тренировка отменена» и блокировать запись (FR-19).
- **Ошибка получения данных**: сообщение об ошибке с возможностью повтора.

## Ограничения и особые случаи
- Все данные — только просмотр, изменение расписания или тренировки с
  этого экрана невозможно (NFR-14).
- Данные должны быть актуальными на момент показа (NFR-6).

## Связанные требования
- Бизнес-требования: BR-4.
- Функциональные требования: FR-3.
- Пользовательские истории: US-2.
- Use Case: UC-2.

## Открытые вопросы для дизайна
- Нужно ли показывать рейтинг инструктора (по итогам оценок из UC-8) на
  этом экране? В требованиях явно не описано как обязательное — можно
  вынести в дизайн как желательное дополнение (Should), согласовав отдельно.
