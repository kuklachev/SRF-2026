# SCR-1. Список тренировок (расписание)

## Назначение экрана
Главный экран приложения. Клиент видит список доступных тренировок и может
перейти к подробной информации о любой из них либо к записи.

## Роль пользователя
Клиент.

## Предусловия
- Приложение запущено.
- Доступен backend (единственный источник данных о тренировках, инструкторах
  и свободных местах — BR-11).

## Точки входа
- Открытие приложения (стартовый экран / нижняя навигация, вкладка
  «Расписание»).
- Возврат с экрана фильтров (SCR-2) с применёнными фильтрами.
- Возврат с экрана подробностей тренировки (SCR-3).

## Состав информации на экране
Для каждой карточки тренировки в списке отображаются (FR-2):
- дата;
- время начала;
- продолжительность;
- формат тренировки (в т.ч. пометка «для новичков», если применимо, —
  влияет на вместимость 8 vs 16, см. бизнес-ограничения);
- инструктор (имя, фото — справочно, без возможности редактирования);
- стоимость;
- количество свободных мест из общей вместимости (например, «5 из 16»).

Список должен явно визуально отличать тренировки:
- с малым числом свободных мест (например, ≤ 2) — сигнал срочности;
- без свободных мест — статус «Мест нет», запись недоступна (FR-8, UC-4
  альтернативный сценарий);
- отменённые скалодромом — статус «Отменена» (см. SCR-8), такие тренировки
  не должны быть доступны для записи (FR-19).

## Действия пользователя
- Прокрутка списка тренировок.
- Переход к настройке фильтров (открывает SCR-2) — FR-4.
- Тап по карточке тренировки → переход на SCR-3 (подробная информация).
- Обновление списка (pull-to-refresh) для получения актуальных данных.
- Переход к разделу «Мои бронирования» (навигация на SCR-6).

## Состояния экрана
- **Загрузка**: skeleton-плейсхолдеры карточек, экран должен становиться
  интерактивным не более чем за 2 секунды при стабильном соединении (NFR-1).
- **Пустое состояние**: нет тренировок, соответствующих текущим фильтрам —
  дружелюбное сообщение и предложение сбросить фильтры.
- **Ошибка**: backend недоступен — сообщение об ошибке с кнопкой повтора
  запроса; список не должен показывать устаревшие/неподтверждённые данные
  как актуальные (NFR-6).
- **Активные фильтры**: индикатор того, что применены фильтры (например,
  бейдж с количеством активных фильтров рядом с иконкой фильтра).

## Ограничения и особые случаи
- Данные всегда должны быть актуальными на момент отображения — экран не
  кэширует расписание как источник истины дольше времени сессии (NFR-6).
- Экран не должен позволять создавать, редактировать или удалять тренировки
  (NFR-14) — только просмотр.
- Верстка без горизонтальной прокрутки, адаптирована под экраны смартфонов
  (NFR-11).

## Связанные требования
- Бизнес-требования: BR-4, BR-11.
- Функциональные требования: FR-1, FR-2, FR-4.
- Пользовательские истории: US-1, US-3.
- Нефункциональные требования: NFR-1, NFR-6, NFR-9, NFR-11, NFR-13.
- Use Case: UC-1, UC-3.

## Открытые вопросы для дизайна
- Нужна ли на этом экране лента баннеров/акций? (Не описано в требованиях —
  предполагается, что нет, экран должен быть минималистичным.)
- Требуется ли отображение уже применённого клиентом «Мои бронирования»
  статуса прямо в карточке (например, бейдж «Вы записаны»)? Уточнить у
  аналитика, в требованиях явно не описано.
